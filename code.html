Network Programming: server/client

Fun Fact: This project was over 3,500 Lines of code!(A glimpse preview)

Client Requirments:

  • All information the user needs is presented to them (i.e. health and status after battles, players entering the room they are in, messages, etc). The user is not be expected to remember things like their own health, nearby monsters, etc. So the client does track these things. The player does not have to be aware of technical details such as the types of lurk messages or the bit order of the player flags.
  • The user is required to type, incoming information does not disturb the typing
  • Client supports the full Lurk protocol. Any server can communicate with my client.

    • Server Requirments:

  • Supports the full Lurk protocol. Messages work as expected..
  • Must supports multiple clients at once(tested), which will interact with each other
  • Map must contains 10 rooms
  • Map contains at 10 entities
  • Server state is persistent. So if a player enters the game and dies, the body remains visible to other players.

    • COMPILE SERVER USING: g++ -o server server.cpp -pthread

    Controlling keyboard LEDs
